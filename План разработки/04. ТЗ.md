## 2. Цели и задачи

### 2.1 Главная цель
Создать AI-агента, который автономно выходит в интернет и социальные сети, собирает трендовую и актуальную информацию по заданным темам, анализирует её и присылает структурированный отчёт пользователю в Telegram.

### 2.2 Задачи агента

1. **Маркетинговая разведка** — поиск актуальной информации для проведения маркетинговых кампаний по продукту или услуге
2. **Анализ трендов** — определение что сейчас в тренде, почему залетело, какой хук или триггер сработал
3. **Подбор контента** — нахождение похожего трендового контента как основы для создания собственного
4. **Генерация идей** — на основе найденного предлагать идеи для постов, видео, Reels
5. **Создание контента** *(расширенный режим)* — запуск цепочки создания контента на основе трендов

`[ + ]` — *новые задачи по мере их появления*

### 2.3 Критерии успеха
- Агент запускается и возвращает отчёт без ошибок в обоих режимах (авто и ручном)
- Отчёт содержит актуальные тренды с объяснением причин виральности
- Время работы одного цикла разведки — не более 10 минут
- При недоступности сервиса агент автоматически переключается на альтернативу

`[ + ]` — *дополнительные метрики качества*

## 3. Режимы работы

### 3.1 Автоматический режим
- Запускается **1 раз в сутки** по расписанию без участия пользователя
- Время запуска настраивается через `/settings` (по умолчанию: 08:00)
- Результат автоматически отправляется в Telegram

### 3.2 Ручной режим
- Запуск по команде пользователя в любое время
- Быстрый поиск по теме: `/search [тема]`
- Внеплановый полный отчёт: `/report`

`[ + ]` — *новые режимы: мониторинг конкурентов, алерты на хайп и др.*

### 3.3 Выбор задачи при запуске

При старте агент предлагает:
- **1. Поиск и анализ контента** — разведка трендов и актуальной информации
- **2. Создание контента** — поиск трендов + запуск цепочки создания

## 7. Интеграция с Marketing agency

### 7.1 Место агента в системе

```
Marketing agency (главный проект)
        ↓ задача
  Главный оркестратор
        ↓
  Агент-разведчик (этот проект)
        ↓ отчёт
  Контент-завод → создание контента
```

### 7.2 Механизм интеграции
- **Протокол: MCP** (Model Context Protocol) — решено в `05. Архитектура.md` разд. 12
- Агент-разведчик экспортирует MCP-методы: `intelligence.search()`, `intelligence.trends()`, `intelligence.analyze()`, `intelligence.status()`
- Агент принимает задачи от главного оркестратора Marketing agency и от пользователя напрямую через Telegram — оба варианта запуска поддерживаются

`[ + ]` — *новые точки интеграции и MCP-методы*

### 7.3 Формат обмена данными
- **Входящий запрос:** текстовая задача + параметры (тема, платформа, тип контента)
- **Исходящий результат:** структурированный отчёт + список идей + найденные материалы

## 8. Нефункциональные требования

### 8.1 Производительность
- Стандартный цикл разведки: не более **10 минут**
- Быстрый режим `/trends`: не более **2–3 минут**
- Во время работы агент присылает статус "работаю..." чтобы не блокировать диалог

### 8.2 Надёжность
- При сбое сервиса — автоматический fallback на альтернативу
- При полном сбое — уведомление пользователя с описанием проблемы
- Логирование всех ошибок

### 8.3 Стоимость
- При равном качестве — выбирается более дешёвый сервис
- Мониторинг расходов на API
