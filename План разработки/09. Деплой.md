## 2. Подготовка сервера (первый деплой)

### 2.1 Минимальные требования сервера

| Параметр | MVP | Рост (v1.5) |
|---|---|---|
| CPU | 2 ядра | 4 ядра |
| RAM | 4 GB | 8 GB |
| Диск | 20 GB SSD | 50 GB SSD |
| ОС | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS |
| Порты открыты | 22 (SSH) | 22, 80, 443 (HTTPS) |

### 2.2 Последовательность настройки сервера

```bash
# 1. Обновить систему
apt update && apt upgrade -y

# 2. Установить Docker
curl -fsSL https://get.docker.com | sh

# 3. Добавить пользователя в группу docker
usermod -aG docker $USER

# 4. Установить Docker Compose
apt install docker-compose-plugin -y

# 5. Проверить установку
docker --version
docker compose version

# 6. Установить Tailscale (VPN)
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up

# [ + ] дополнительные шаги настройки сервера
```

### 2.3 Установить Portainer (управление контейнерами)

```bash
docker volume create portainer_data

docker run -d \
  -p 8000:8000 \
  -p 9443:9443 \
  --name portainer \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer-ce:latest
```

После запуска — открыть в браузере через Tailscale: `https://<server-ip>:9443`

## 3. Деплой агента (MVP)

### 3.1 Порядок первого деплоя

```
1. Подключиться к серверу по SSH
   ssh root@<hetzner-ip>

2. Склонировать репозиторий
   git clone https://github.com/<username>/intelligence-agent.git
   cd intelligence-agent

3. Создать .env файл
   cp .env.example .env
   nano .env      ← заполнить все API-ключи

4. Запустить все контейнеры
   docker compose up -d

5. Проверить статус контейнеров
   docker compose ps

6. Проверить логи
   docker compose logs -f

7. Отправить /start боту в Telegram — должен ответить
```

### 3.2 Проверка после деплоя

- [ ] Все контейнеры в статусе `running`
- [ ] Telegram Bot отвечает на `/start`
- [ ] `/report` запускается и возвращает результат
- [ ] Логи не содержат критических ошибок
- [ ] Redis доступен: `docker exec -it redis redis-cli ping` → PONG
- [ ] Supabase подключён: тест через Telegram `/report`

`[ + ]` — *дополнительные проверки*

## 4. Обновление агента (последующие деплои)

### 4.1 Стандартный деплой обновления

```bash
# На сервере:
cd intelligence-agent

git pull origin main             # забрать изменения

docker compose pull              # обновить образы (если изменились)

docker compose up -d --build     # пересобрать и перезапустить

docker compose ps                # проверить статус
```

### 4.2 Деплой без остановки (нулевой даунтайм)

```bash
# Обновить только один сервис без остановки остальных
docker compose up -d --no-deps --build <service-name>

# Пример — обновить только MCP-сервер Apify:
docker compose up -d --no-deps --build mcp-apify
```

### 4.3 Откат при проблемах

```bash
# Откатиться на предыдущую версию через git
git log --oneline -5             # посмотреть последние коммиты
git checkout <commit-hash>       # переключиться на предыдущий коммит
docker compose up -d --build     # перезапустить с предыдущей версией
```

```bash

# Логи конкретного контейнера
```

## 7. Безопасность продакшн-сервера

### 7.1 Базовые меры

- [ ] SSH только по ключу, отключить вход по паролю
- [ ] Firewall (ufw): открыть только нужные порты (22, 80, 443)
- [ ] Tailscale для Portainer — не открывать порт 9443 публично
- [ ] `.env` не попадает в git (проверить `.gitignore`)
- [ ] Ротация API-ключей раз в 3–6 месяцев
- [ ] Telegram Bot whitelist: только `TELEGRAM_ALLOWED_USER_IDS`
