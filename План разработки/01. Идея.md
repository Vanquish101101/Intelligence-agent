# 01. Идея — Агент онлайн-разведчик

## Суть идеи

Создать AI-агента («Разведчик»), который автономно мониторит интернет и социальные сети, собирает трендовую и актуальную информацию, затем формирует отчёт и аналитику для пользователя.

Агент работает **по расписанию (1 раз в день)** или **по запросу пользователя** — всегда в курсе трендов в выбранных сферах.

---

## Режим работы

| Режим | Описание |
|---|---|
| Автоматический | Запуск 1 раз в сутки без участия пользователя |
| Ручной | Запуск по запросу пользователя в любое время |

### Глубина анализа

| Уровень | Модель | Инструменты | Время |
|---|---|---|---|
| `quick` | claude-haiku-4-5 | Perplexity sonar, краткий отчёт | 15–30 сек |
| `standard` | claude-haiku-4-5 | Все инструменты, стандартный объём | 60–90 сек |
| `deep` | claude-sonnet-4-6 | Perplexity sonar-pro + расширенный сбор | 2–5 мин |

**Правило:** все запросы на анализ и поиск выполняются в режиме `standard` по умолчанию. При выборе `deep` включается deepresearch-режим с максимальной детализацией.

## Задачи агента

1. **Маркетинговая разведка** — поиск актуальной информации для проведения маркетинговых кампаний по продукту или услуге.
2. **Анализ трендов** — подбор похожего трендового контента как основы для создания собственного.
3. **Генерация контента** *(на основе найденного)* — посты, фото, видео, Reels при помощи специализированных сервисов.
4. **OSINT-разведка** — поиск по открытым источникам: никнейм, домен, персона, компания, email, телефон, IP. Использует Perplexity sonar-pro + Firecrawl. Команда `/osint [тип] цель`.
5. **Учёт затрат** — агент анализирует расходы на API и выводит таблицу трат после каждого запроса. Команда `/costs`.

> Список задач может дополняться и уточняться в ходе разработки.

## Алгоритм работы (концепция)

```
Запрос (пользователь / расписание)
        ↓
  Оркестратор
  - Понимает задачу
  - Выбирает ресурсы (по качеству и стоимости)
  - Управляет взаимозаменяемостью ресурсов
        ↓
  Агент-разведчик
  - Парсинг площадок и соц. сетей
  - Сбор контента по теме
  - Анализ трендов
        ↓
  Формирование отчёта
  - Аналитика
  - Подборка материалов
  - Рекомендации
        ↓
  Отправка пользователю
```

**Ключевые принципы оркестратора:**
- Понимает задачу и процессы её выполнения
- Выбирает оптимальные ресурсы по соотношению качество/стоимость
- При недоступности ресурса — автоматически переключается на альтернативный
- Управляет всеми агентами, плагинами и скиллами

## Связь с основным проектом

Агент является **вспомогательным модулем** основного проекта **Marketing agency (Маркетинговое агентство)** и выполняет задачи для **Контент-завода**:

- Получает задачи от главного оркестратора проекта
- Возвращает результаты разведки обратно в систему
- Интеграция через **MCP-серверы** или специализированные сервисы *(уточняется)*

```
Marketing agency (главный проект)
        ↓ задача
  Главный оркестратор
        ↓
  Агент-разведчик (этот проект)
        ↓ отчёт
  Контент-завод → создание контента
```
