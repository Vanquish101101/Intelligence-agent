## 5. Версионирование и Git-конвенции

### 5.1 Стратегия веток

```
main          ← стабильная версия (только merge из dev)
dev           ← активная разработка
feature/xxx   ← отдельная фича (merge в dev по готовности)
fix/xxx       ← исправление бага
```

### 5.2 Формат коммитов

```
feat: добавить MCP-сервер для Apify
fix: исправить timeout при парсинге TikTok
refactor: разделить оркестратор на модули
docs: обновить описание реестра инструментов
chore: обновить зависимости Node.js
```
