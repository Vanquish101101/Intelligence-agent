## 1. Идеи и новые возможности агента

### Основные идеи (из концепции проекта)
- Агент работает в двух режимах: поиск/анализ контента и создание контента
- Автозапуск 1 раз в сутки без участия пользователя
- Запуск по запросу в любое время через Telegram
- Отчёт и аналитика приходят прямо в Telegram в читаемом виде
- Мониторинг соц. сетей: Telegram, VK, YouTube, TikTok, Одноклассники, WhatsApp

### Дополнительные идеи (предложения)
- **Алерты на хайп в реальном времени** — если что-то резко набирает популярность, агент присылает уведомление не дожидаясь расписания
- **Мониторинг конкурентов** — отслеживать конкретные аккаунты или каналы конкурентов и анализировать их контент
- **Персональные темы-фильтры** — пользователь задаёт список тем (крипта, маркетинг, партнёрки) и агент ищет только по ним
- **История отчётов** — сохранение всех прошлых отчётов, чтобы видеть динамику трендов во времени
- **Сравнение периодов** — "что было в тренде на этой неделе vs прошлой"
- **Оценка виральности** — агент не просто находит контент, но и объясняет ПОЧЕМУ он залетел (хук, триггер, эмоция, тайминг)
- **Мультиязычный поиск** — поиск не только на русском, но и на английском для поиска глобальных трендов
- **Голосовые сводки** — отчёт в виде аудио (через TTS) для прослушивания на ходу
- **Интеграция с контент-планом** — найденные тренды автоматически попадают в очередь контент-завода
- **Рейтинг источников** — агент со временем учится каким источникам доверять больше

### Трёхуровневая цепочка обработки данных

```
УРОВЕНЬ 1 — СБОР (дёшево, много)
  Apify           → парсинг соц. сетей: Telegram, YouTube, VK, TikTok
  Firecrawl       → парсинг сайтов, статей, текстового контента

УРОВЕНЬ 2 — ФИЛЬТРАЦИЯ (проверка, транскрибация)
  Perplexity      → проверка фактов, бизнес-разведка
  Whisper         → транскрибация видео в текст

УРОВЕНЬ 3 — АНАЛИЗ (смыслы, хуки, итог)
  Chat GPT 5.5    → понимание смыслов, извлечение хуков, финальный отчёт
  Open Router     → маршрутизирует между моделями, выбирает дешевле/лучше
```

### Принцип взаимозаменяемости (решено)
- У каждого инструмента есть 1–2 запасных в YAML-реестре (`fallback`)
- Health check каждые 5 минут — система сама переключается до сбоя
- Новый инструмент = новый контейнер + запись в реестр, без остановки системы
- Open Router выбирает дешёвую модель для рутины и GPT 5.5 только для глубокого анализа

## 3. UX и взаимодействие с пользователем

### Текущая концепция
- Всё взаимодействие через **Telegram бот**
- Запуск: `/start` → выбор режима → результат в чат
- Автоматический отчёт раз в день без действий пользователя

### Идеи по UX

**Команды бота:**
- `/start` — старт и выбор режима
- `/search [тема]` — быстрый поиск по теме прямо из чата
- `/report` — запросить внеплановый отчёт прямо сейчас
- `/settings` — настройки: темы, соц. сети, расписание, язык
- `/history` — история последних отчётов
- `/trends` — что сейчас в тренде (быстрая сводка)

**Формат отчёта в Telegram:**
```
📊 ОТЧЁТ РАЗВЕДЧИКА — [дата]
━━━━━━━━━━━━━━━━━━━━━
🔥 ТОП ТРЕНДЫ СЕГОДНЯ:
1. [тема] — почему залетело: [объяснение]
2. [тема] — хук: [что зацепило]

📱 ПО ПЛАТФОРМАМ:
• YouTube: [топ видео]
• TikTok: [топ контент]
• Telegram: [горячие каналы]

💡 ИДЕИ ДЛЯ КОНТЕНТА:
• [идея 1]
• [идея 2]

⚠️ ВАЖНО ПРОВЕРИТЬ:
• [открытый вопрос или риск]
━━━━━━━━━━━━━━━━━━━━━
```

**Настройки пользователя:**
- Темы мониторинга (список ключевых слов / ниш)
- Какие соц. сети включить/выключить
- Время автоотчёта (например, 08:00 каждый день)
- Язык поиска (RU / EN / оба)
- Глубина анализа (быстро / подробно)

## 4. Контентная стратегия

### Текущие задачи агента
- Поиск трендового контента для маркетинговых кампаний
- Анализ и подбор похожего контента как основы для своего
- Понимание что именно вызвало вирусный эффект

### Ниши и темы (из контекста проекта)
- Партнёрские программы и заработок онлайн
- Криптовалюта и инвестиции
- Маркетинг и продвижение в соц. сетях
- *(список пополняется)*

### Идеи по контентной стратегии
- **Матрица тем** — для каждой ниши свой список источников и ключевых слов
- **Сезонность** — агент учитывает сезонные тренды (праздники, события, инфоповоды)
- **Анти-тренды** — что перестало работать, чего избегать
- **Хуки-шаблоны** — агент извлекает успешные хуки и сохраняет как шаблоны для будущего контента
- **Геотаргетинг** — тренды по регионам (Россия / СНГ / мир)
- **Конкурентный анализ** — что публикуют лидеры ниши

### Типы контента на выходе (после анализа)
- Пост для Telegram / VK
- Сценарий для Reels / TikTok
- Идея для YouTube видео
- Заголовок и структура статьи

## 5. Монетизация и масштаб

### Текущий контекст
- Агент создаётся для личного использования внутри Marketing agency
- Является модулем Контент-завода

### Идеи по монетизации (на будущее)
- **White-label продукт** — упаковать агента как отдельный инструмент для продажи маркетологам
- **SaaS подписка** — другие агентства платят за доступ к агенту-разведчику
- **Консалтинг на основе данных** — продавать не агента, а отчёты и аналитику как услугу
- **Партнёрская модель** — агент находит партнёрские программы → автоматически встраивает в контент
- **Масштабирование** — запуск нескольких агентов по разным нишам параллельно

### Потенциальные клиенты агента
- Маркетинговые агентства
- SMM-специалисты и контент-мейкеры
- Инфобизнес и онлайн-школы
- Крипто и финансовые проекты

## 6. Риски и слабые места

### Технические риски
| Риск | Вероятность | Последствие | Решение |
|---|---|---|---|
| Apify заблокирован соц. сетью | Средняя | Нет данных по платформе | Альтернативный парсер через Smithery |
| Превышение лимитов API | Высокая при росте | Дорого или остановка | Кэширование, лимиты, очереди |
| Chat GPT 5.5 недоступен | Низкая | Агент не работает | Fallback на другую модель через Open Router |
| Firecrawl не может спарсить сайт | Средняя | Нет данных с источника | Jina AI или ручная ссылка |
| Устаревание данных | Высокая | Неактуальные тренды | Частота обновления + временные метки |

### Бизнес-риски
- Зависимость от сторонних API — если сервис закрывается, часть функций пропадает
- Стоимость API может вырасти при масштабировании
- Соц. сети меняют алгоритмы и блокируют парсинг
- Качество отчётов зависит от качества промптов — нужна постоянная настройка

### Слабые места текущей схемы
- Роль Firecrawl в видео-цепочке пока не до конца определена
- Механизм взаимозаменяемости сервисов не описан технически
- Нет механизма хранения истории и обучения на прошлых отчётах
- WhatsApp крайне сложен для парсинга (закрытая платформа)
